# RULES.md

## Regra: Documentação

- O projeto **NÃO deve conter múltiplos arquivos `.md`**.
- São permitidos **apenas dois arquivos Markdown**:
  - `README.md`
  - `RULES.md` (este arquivo)
- Qualquer outra documentação deve estar **dentro do README ou do RULES**.

---

## Regra: Imutabilidade do RULES

- O arquivo `RULES.md` **NÃO deve ser alterado**.
- O `RULES.md` deve **apenas ser seguido**, nunca modificado, ajustado ou reescrito durante o projeto.



---

## Regra: Organização do Projeto

- Todo o projeto deve ser **totalmente organizado em pastas**.



---

## Regra: Inicialização do Projeto

- Todo projeto deve possuir um arquivo `.bat` para iniciar o projeto.



---

## Regra: Rodapé do Projeto

- Todos os projetos devem conter no rodapé a identificação:
  - **Dev Aleksandro Alves**



---

## Regra: Sugestões de Melhoria

- Sempre devem ser sugeridas melhorias ao projeto.
- As melhorias devem ser apresentadas em formato de **menu de opções**.
- Apenas a melhoria escolhida pelo usuário deve ser implementada.



---

## Regra: Modificações Pontuais

- Quando for solicitada uma modificação, deve ser alterado **apenas** o que foi explicitamente pedido.
- Nenhuma outra parte do projeto deve ser modificada sem autorização direta.
